## Approach

Construct a bare `Device` (no DataItems for test 1; one user-declared
`AssetCountDataItem` with `Representation = VALUE` for test 2), call
`MTConnectAgent.AddDevice(device, initializeDataItems: false)`, and read
back the resulting DataItem's `Representation`. The
`initializeDataItems: false` flag avoids the timer-based observation
machinery — the test only needs to inspect the device structure
returned from `NormalizeDevice`.

`MTConnectAgent` is constructed with `initializeAgentDevice: false` to
avoid creating the implicit Agent device (it is unnecessary for this
test and adds noise to inspection).

## Spec citation

MTConnect Standard, Part 2 — Devices Information Model, defines
`ASSET_COUNT` (UML element ID
`_19_0_3_68e0225_1640602520420_217627_44`, captured in
`AssetCountDataItem.g.cs`) as a `DATA_SET` representation per the SysML
model. The `.g.cs` source comment is the canonical pointer.
